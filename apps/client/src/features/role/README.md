# Role Select (P0-01)

## Задача

Экран выбора роли (М/Ж) для старта core flow и единовременного сохранения
выбора в клиенте.

## Требования

- Показывает две крупные кнопки выбора роли.
- Не содержит сетевой логики, только вызывает `onSelect`.
- UI простая, с фокусом на кнопках.
- Отображается только если роль еще не сохранена локально.
- После выбора роль сохраняется и больше не запрашивается повторно.

## Обоснование

Изолируем выбор роли как отдельную вертикальную фичу, чтобы дальнейшие шаги
(очередь/мэтч) подключались без переписывания UI.
